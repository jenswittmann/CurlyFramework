CurlyFramework Personal framework for more accessibility and sustainability, not just for MODX.

Installation: $ npm i curlyframework & Github

Better accessibility and sustainability in web development?

It doesn't need a complete relaunch, small steps can also help to reach the goal. I recommend the talk by Michelle Barker "Building a greener web" at "All Day Hey! 2023".

Tips

  • Use WebP and lazyload (images, embeds).
  • Use less fonts and try to use variable fonts (.woff2).
  • Only load embeds on user interaction.
  • Test keyboard navigation.
  • Seek community to test and discuss accessibility and sustainability with real people.

Simple Answer for the Accordion.

All elements scale if desired via the class .scale-content in the HTML node proportionally. Together with the CSS Grid and the classes for the responsive font a super basis for design presets from Figma (Adobe XD) and zeplin.io.

The .htaccess has been extended with caching and security settings. Together with ContentBlocks from modmore, you can create a flexible website builder with 100% code control. The file modx.css brings a minimalistic layout for the MODX Manager backend.

Installation

  • Get it via $ git clone git@github.com:jenswittmann/CurlyFramework.git or use $ npm i curlyframework (as npm dependency).
  • Link CSS/JS files in your markup.
  • Overwrite CSS via custom properties and use inline JS via AlpineJS.

Need more?

  • Install dependency $ npm i
  • Use CodeKit or esbuild (need custom config) to compile your assets.
  • Play around, learn and share.
